# Аналитика студенческих документов — план

> Статус: идея, отложено
> Дата: 2026-04-06
> Контекст: в Supabase Storage накопилось 2,154 документов (868 MB) + 1,319 результатов (641 MB)

## Гипотеза

Загруженные документы (курсовые, дипломы, рефераты) и методички содержат структурированные метаданные на титульных страницах, которые можно извлечь и использовать для маркетинга, продукта и конкурентного преимущества.

## Что извлекать

С титульных страниц (1-2 страница документа):
- **ВУЗ** — полное название учебного заведения
- **Факультет / кафедра** — специальность
- **Тип работы** — курсовая, дипломная, ВКР, реферат, отчёт по практике
- **Дисциплина** — если указана
- **Город** — география
- **Год**

**НЕ извлекать:** ФИО студентов, преподавателей, номера групп (персональные данные).

## Применение данных

### 1. Таргетированный маркетинг (высокая ценность)
- Топ-10 ВУЗов по количеству пользователей -> таргетированная реклама в их VK-группах, Telegram-чатах
- Понимание сезонности по ВУЗам (сессии, защиты)
- Региональная реклама (если Москва/Питер доминируют — или наоборот)

### 2. SEO-страницы (высокая ценность)
- Создать landing pages: "оформление курсовой по экономике", "дипломная работа ГОСТ юриспруденция"
- Ключевые слова из реальных специальностей пользователей
- Контент для блога на основе реальных потребностей

### 3. Автоматические шаблоны по ВУЗам (убийственная фича)
- Накопить базу методичек -> "Выбери свой ВУЗ -> мы уже знаем требования"
- Конкурентное преимущество, которое сложно скопировать
- Методички уже загружаются пользователями (PDF в documents bucket)

### 4. Приоритизация продукта (средняя ценность)
- Какие типы работ доминируют -> на что фокусировать фичи
- Какие специальности -> какие шаблоны цитирования нужны
- Средний размер документа -> оптимизация лимитов

### 5. Понимание рынка (средняя ценность)
- Сколько ВУЗов охвачено
- Потенциал роста (сколько ещё ВУЗов в России)
- B2B возможности (предложить сервис самому ВУЗу)

## Техническая реализация

### Фаза 1: Извлечение из текущих файлов (одноразово)
1. Скачать первые 2 страницы каждого DOCX/PDF из `documents` bucket
2. Прогнать через AI (Gemini Flash Lite — бесплатный) с промптом на извлечение метаданных
3. Сохранить в таблицу `document_metadata` в Supabase (~100 KB вместо 868 MB)
4. Удалить оригинальные файлы из Storage

### Фаза 2: Автоматическое извлечение (на будущее)
1. При загрузке документа — извлечь метаданные до обработки
2. Сохранить в `document_metadata`
3. Не хранить оригинал после обработки (настроить TTL на файлы)

### Фаза 3: Аналитический дашборд
1. Топ ВУЗов (pie chart)
2. Топ специальностей
3. Типы работ
4. Динамика по неделям
5. География

## Таблица document_metadata (схема)

```sql
CREATE TABLE document_metadata (
  id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  job_id text,                    -- связь с jobs таблицей
  university text,                -- название ВУЗа
  faculty text,                   -- факультет/кафедра
  specialty text,                 -- специальность
  work_type text,                 -- курсовая/диплом/ВКР/реферат
  discipline text,                -- дисциплина
  city text,                      -- город
  year integer,                   -- год
  is_methodology boolean,         -- это методичка?
  file_type text,                 -- docx/pdf
  file_size integer,              -- размер файла
  created_at timestamptz DEFAULT now()
);
```

## Privacy / юридические аспекты

- НЕ сохранять ФИО, номера групп, контакты
- Обновить Privacy Policy: "метаданные документов используются для улучшения сервиса"
- Данные агрегированные, не привязаны к конкретному пользователю
- При извлечении — только структурные поля, не содержание работы

## Приоритет

Средний. Не блокирует основной продукт, но может дать:
- Инсайты для маркетинга (CPO)
- Основу для killer-фичи с шаблонами по ВУЗам (продукт)
- SEO-контент на основе реальных данных (рост)
