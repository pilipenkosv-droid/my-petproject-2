SmartFormat Product Plan

План развития SmartFormat — Продуктовая стратегия

Стратегическое видение

Миссия: Стать стандартом автоматического форматирования студенческих работ в России, используя уникальное преимущество — загрузку методички вуза.

North Star Metric: Количество полностью обработанных документов / месяц

Ресурсы: Время основателя + Claude Code Max. Маркетолог на доле 70/30%.

Главный принцип: Не конкурируем по цене с бесплатными сервисами. Конкурируем по ценности — кастомизация под методичку вуза + библиотека требований вузов.

Бесплатные каналы привлечения (приоритизированные)

1. SEO Content Hub (Приоритет: МАКСИМАЛЬНЫЙ)

Почему: Долгосрочный, компаундный рост. Каждая статья работает 24/7. - 15 статей уже есть → довести до 30 за 3 месяца - Фокус на long-tail запросы с низкой конкуренцией - Каждая статья = воронка в продукт

2. Бесплатные инструменты как growth-hack (Приоритет: ВЫСОКИЙ)

Почему: Привлекают трафик + показывают ценность без оплаты. - /outline — генерация плана (бесплатный, безлимитный) - /summarize — сумматор (1 бесплатный) - /rewrite — рерайт (только для платных)

Воронка:

Студент ищет "план курсовой работы"
  → Находит /outline в Google/Yandex
  → Генерирует план бесплатно
  → Видит CTA: "У вас уже есть работа? Отформатируем по ГОСТу"
  → Переходит на /create

Чат с методичкой — детали

Как работает: 1. Пользователь загружает методичку → текст уже парсится в guidelinesText 2. На странице /confirm-rules/[jobId] появляется кнопка “Спросить про методичку” 3. ПРОВЕРКА ПЛАНА

Как валидировать каждую фазу

  1. Фаза 0: Metrika/GA работают, воронка отображается → переходим к M1
  2. Неделя 1: CR free→paid замерен до и после улучшения hook-offer
  3. Неделя 2: >30% пользователей используют чат с методичкой
  4. Неделя 3: /outline генерирует >100 планов/мес, >10% переходят на /create
  5. Неделя 4: Новые статьи начинают индексироваться (Webmaster)
  6. M2: Органический трафик >800, MRR >25К₽
  7. M3: Органический трафик >2000, MRR >60К₽, готовность к сезону защит

E2E тестирование

  • Пройти путь пользователя: лендинг → /create → загрузка → результат → оплата → скачивание
  • Проверить каждый новый инструмент: /outline, /summarize, /rewrite
  • Проверить SEO: все страницы в индексе, метаданные корректны, JSON-LD валиден
